Hidden bug

A real data race was masked by the go.work failure

cmd/md-go-validator/main_test.go — TestHandleVersion

Because test-race couldn't even run (go.work blocked it), a genuine data race went undetected. TestHandleVersion mutated three process globals (os.Stdout, osExit, version) while running under t.Parallel() — racing with ~15 other parallel tests that call printUsage() or run the CLI. The sibling test TestParseArgsVersionFlag already had the correct pattern (serial, with //nolint:paralleltest), so TestHandleVersion was an oversight, not a design choice.

Lesson: When a gate can't run, it can't catch bugs. The go.work failure created a blind spot that hid a concurrency defect for an unknown period.
False positives

"Executable in repo" warnings on gitignored files

dist/*, result — confirmed untracked via git ls-files

BuildFlow flagged 6 binaries in dist/ and a result symlink as "compiled binaries committed to version control". They are not tracked — .gitignore covers /dist/ and /result. The lint tool scans the filesystem, not git. These are noise, not real issues.

EWhat We Should Improve

1. Never let a gate be un-runnable

The go.work failure disabled 6 quality gates simultaneously. A single missing workspace entry created a blind spot that hid a data race. The fix is process-level: any gate that can't run should fail loud and block everything, not silently produce "failed to wait for command" errors that look like 6 different problems.

2. Eliminate global mutation in cmd/main.go

The data race root cause is architectural: cmd/md-go-validator/main.go uses package-level globals (version, osExit, os.Stdout) that tests must mutate. This forces serial tests and //nolint:paralleltest hacks. The proper fix is dependency injection: pass an io.Writer and an exit function into a struct, so tests never touch globals.

3. Populate the missing standard doc files

FEATURES.md, TODO_LIST.md, ROADMAP.md are all absent. docs/DOMAIN_LANGUAGE.md is a placeholder. These are defined in the project documentation standards as required files with distinct purposes. Their absence means project state is invisible without reading code.

4. Make the go-error-family decision explicit

92 hierarchical-errors findings (45 of them generic_return) will keep failing BuildFlow until either (a) go-error-family is adopted, or (b) the check is explicitly disabled/acknowledged. This is a deliberate architectural choice that should be documented as an ADR rather than left as a persistent red gate.

5. Add the 2 silent_swallow findings to the backlog

pkg/validator.go:431 discards a context error with a bare break. pkg/validator.go:412 sends to an error channel but the checker flags the non-returning pattern. These are benign for concurrent worker code but should be explicitly acknowledged or refactored.
